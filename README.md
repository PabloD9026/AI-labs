# Ход работы
## Начальная обработка данных ([Код](eda.ipynb))
Этот проект объединяет несколько наборов данных об автомобилях в один согласованный файл, кодирует категориальные признаки, нормализует числовые значения и разделяет данные на обучающую и тестовую выборки. Ниже подробно описан каждый шаг и используемые библиотеки.

## Используемые библиотеки

- **pandas** – загрузка, обработка и объединение данных.
- **scikit-learn** – для LabelEncoder (кодирование меток), train_test_split (разделение на обучающую/тестовую выборки) и MinMaxScaler (масштабирование).
- **numpy**, **matplotlib**, **seaborn** – (опционально, показаны в исследовательском анализе) для численных операций и визуализации.

## Пошаговый процесс

### 1. Объединение данных

Загружаются три исходных CSV-файла:
- `Car_data.csv`
- `Car_details_v3.csv`
- `CAR_DETAILS_FROM_CAR_DEKHO.csv`

Первый набор данных (`Car_data.csv`) имеет другие названия столбцов, поэтому его столбцы переименовываются в соответствии со схемой третьего набора (`CAR_DETAILS_FROM_CAR_DEKHO.csv`). Общие столбцы, которые остаются:

- `name` – модель автомобиля
- `year` – год выпуска
- `selling_price` – цена продажи (в индийских рупиях)
- `km_driven` – пробег в километрах
- `fuel` – тип топлива (бензин, дизель, CNG и т.д.)
- `seller_type` – тип продавца (частное лицо или дилер)
- `transmission` – коробка передач (механика или автомат)
- `owner` – количество предыдущих владельцев

Все три датасета объединяются по строкам в один DataFrame и сохраняются как `merged.csv`.

### 2. Кодирование категориальных признаков

Поскольку модели машинного обучения требуют числовых входных данных, все категориальные столбцы (тип object) преобразуются с помощью **LabelEncoder** (`sklearn.preprocessing.LabelEncoder`). Каждой уникальной категории ставится в соответствие целое число (0, 1, 2, …). Закодированный датасет сохраняется как `encoded.csv`.

### 3. Нормализация

Закодированный датасет нормализуется с помощью **MinMaxScaler** (`sklearn.preprocessing.MinMaxScaler`). Он приводит каждый признак к диапазону [0, 1] по формуле:

X_scaled = (X - X_min) / (X_max - X_min)


Нормализация гарантирует, что ни один признак не будет доминировать над другими из-за разных единиц измерения или масштаба. Полный нормализованный датасет сохраняется как `normalized.csv` – этот файл используется для дальнейшего анализа и визуализации.

### 4. Разделение на обучающую и тестовую выборки

Нормализованный датасет случайным образом перемешивается и разделяется на **80% обучающей** и **20% тестовой** выборок с помощью `train_test_split` и параметра `random_state=42` для воспроизводимости результатов. Разделенные данные сохраняются как:
- `train.csv`
- `test.csv`

## Итоговые файлы

Здесь важно помнить, что после нормализации Automatic может быть закодирован как 1, а Manual как 0 (или наоборот). Знак корреляции говорит о том, что автоматическая трансмиссия связана с более низкой ценой (в данном датасете, вероятно, преобладают бюджетные автомобили с автоматом или выборка смещена). Причина в том, что name после кодирования стал произвольным числом, не несущим информации о классе автомобиля (премиум/бюджетный). Для учёта модели потребовалось бы создавать отдельные категориальные признаки (one-hot encoding).

## Реализация программного кода и обучения моделей

### Организация проекта и DVC
Проект структурирован для обеспечения воспроизводимости экспериментов с использованием **DVC (Data Version Control)**:
* `data/raw/`: Исходные файлы данных.
* `src/`: Модульный код на Python (загрузка, предобработка, обучение).
* `models/`: Сохраненные веса и сериализованные модели.
* `params.yaml`: Конфигурация гиперпараметров для всех этапов пайплайна.

### Создание пайплайна
Для автоматизации процесса обучения настроен файл `dvc.yaml`, включающий следующие стадии:
1. **Preprocess**: Очистка и подготовка данных.
2. **Train**: Обучение пяти типов моделей (Linear, Tree, CatBoost, XGBoost, NN).
3. **Evaluate**: Сбор метрик и построение графиков.

После того как pipeline был построен можно было запускать его при помощю команда
`dvc repro`

В процессе работы были обучены и протестированы следующие алгоритмы:
Linear Regression: Базовая модель для оценки линейных зависимостей.
Decision Tree: Модель дерева решений (построена визуализация первых узлов).
