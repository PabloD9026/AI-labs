## Пошаговый процесс

### 1. Объединение данных

Загружаются три исходных CSV-файла:
- `Car_data.csv`
- `Car_details_v3.csv`
- `CAR_DETAILS_FROM_CAR_DEKHO.csv`

Первый набор данных (`Car_data.csv`) имеет другие названия столбцов, поэтому его столбцы переименовываются в соответствии со схемой третьего набора (`CAR_DETAILS_FROM_CAR_DEKHO.csv`). Общие столбцы, которые остаются:

- `name` – модель автомобиля
- `year` – год выпуска
- `selling_price` – цена продажи (в индийских рупиях)
- `km_driven` – пробег в километрах
- `fuel` – тип топлива (бензин, дизель, CNG и т.д.)
- `seller_type` – тип продавца (частное лицо или дилер)
- `transmission` – коробка передач (механика или автомат)
- `owner` – количество предыдущих владельцев

Все три датасета объединяются по строкам в один DataFrame и сохраняются как `merged.csv`.

### 2. Кодирование категориальных признаков

Поскольку модели машинного обучения требуют числовых входных данных, все категориальные столбцы (тип object) преобразуются с помощью **LabelEncoder** (`sklearn.preprocessing.LabelEncoder`). Каждой уникальной категории ставится в соответствие целое число (0, 1, 2, …). Закодированный датасет сохраняется как `encoded.csv`.

### 3. Нормализация

Закодированный датасет нормализуется с помощью **MinMaxScaler** (`sklearn.preprocessing.MinMaxScaler`). Он приводит каждый признак к диапазону [0, 1] по формуле:

X_scaled = (X - X_min) / (X_max - X_min)


Нормализация гарантирует, что ни один признак не будет доминировать над другими из-за разных единиц измерения или масштаба. Полный нормализованный датасет сохраняется как `normalized.csv` – этот файл используется для дальнейшего анализа и визуализации.

### 4. Разделение на обучающую и тестовую выборки

Нормализованный датасет случайным образом перемешивается и разделяется на **80% обучающей** и **20% тестовой** выборок с помощью `train_test_split` и параметра `random_state=42` для воспроизводимости результатов. Разделенные данные сохраняются как:
- `train.csv`
- `test.csv`

## 1. Линейная регрессия (Linear Regression)

Линейная регрессионная модель описывает зависимость целевой переменной $y$ от вектора признаков $\mathbf{x} \in \mathbb{R}^p$ в виде линейной комбинации:

$$
y = \beta_0 + \sum_{j=1}^{p} \beta_j x_j + \varepsilon,
$$

где $\beta_0$ — свободный член, $\beta_j$ — коэффициенты модели, а $\varepsilon$ — случайная ошибка с нулевым математическим ожиданием и постоянной дисперсией $\sigma^2$. Оценка параметров обычно производится методом наименьших квадратов путём минимизации суммы квадратов остатков. Основные предположения классической линейной регрессии: линейность связи, независимость ошибок, гомоскедастичность, отсутствие мультиколлинеарности и нормальность распределения ошибок (для статистических выводов). Модель интерпретируема, вычислительно эффективна, но чувствительна к выбросам и не способна улавливать нелинейные зависимости без предварительного конструирования признаков.

## 2. Решающее дерево (Decision Tree)

Решающее дерево — это непараметрическая модель, осуществляющая рекурсивное разбиение пространства признаков на непересекающиеся области, в каждой из которых предсказание определяется простым правилом (например, средним значением для регрессии или наиболее частым классом для классификации). Алгоритм построения: на каждом шаге выбирается признак и порог, максимизирующие уменьшение функции неоднородности (impurity). Для классификации распространены критерии Gini (Джини) и энтропийный критерий, для регрессии — среднеквадратическая ошибка. Полученное дерево обычно подвергается стрижке (pruning) для борьбы с переобучением. Достоинства — интерпретируемость, способность улавливать нелинейные взаимодействия, работа с пропусками; недостатки — высокая вариативность, склонность к переобучению без регуляризации.

- **criterion:** Функция оценки качества разбиения. Значение `squared_error` означает, что в качестве критерия используется среднеквадратическая ошибка (дисперсия в узлах), что характерно для регрессионных деревьев.
- **max_depth:** Максимальная глубина дерева. Установлена в `8` — ограничивает рост дерева для предотвращения переобучения.
- **min_samples_split:** Минимальное количество объектов в узле, необходимое для его дальнейшего разбиения. Значение `3` означает, что узел не будет разделён, если в нём меньше трёх наблюдений.
- **min_samples_leaf:** Минимально допустимое количество объектов в листовом узле. Значение `2` гарантирует, что каждый лист будет содержать не менее двух примеров.
- **output_levels:** Количество отображаемых уровней дерева при визуализации / выводе. Значение `3` предписывает показывать только первые три уровня (корень и два уровня разбиений) для компактности отображения.

## 3. CatBoost (Categorical Boosting)

CatBoost — это метод градиентного бустинга, специально разработанный для эффективной обработки категориальных признаков и борьбы с переобучением. Ключевые особенности:

- Для кодирования категорий применяется упорядоченное целевое кодирование с использованием искусственно создаваемой временной структуры (ordered target statistics), позволяющее избежать «утечки» целевой переменной (data leakage) и переобучения.
- Базовыми моделями являются несимметричные (oblivious) решающие деревья, в которых на каждом уровне используется один и тот же признак и один и тот же порог для всех узлов. Это ускоряет вычисления и устойчиво к шуму.
- В отличие от классического бустинга, CatBoost использует упорядоченный бустинг (ordered boosting), где для каждого объекта градиенты рассчитываются по подвыборке, не содержащей данный объект, что уменьшает смещение градиентных оценок.
- Дополнительно CatBoost автоматически обрабатывает текстовые признаки и пропуски, не требуя предварительной обработки.

Благодаря этим решениям CatBoost обеспечивает высокое качество и стабильность без сложного подбора гиперпараметров.

- **iterations:** Количество итераций бустинга (число деревьев). Значение `100` — строится 100 деревьев.
- **learning_rate:** Темп обучения (шаг градиентного спуска). Значение `0.1` уменьшает вклад каждого дерева, помогая избежать переобучения.
- **depth:** Глубина каждого дерева. Установлена в `8`; CatBoost использует симметричные (oblivious) деревья одинаковой глубины на всех уровнях.
- **l2_leaf_reg:** Коэффициент L2-регуляризации весов в листьях. Значение `3` добавляет штраф за большие значения предсказаний в листьях, уменьшая переобучение.
- **loss_function:** Функция потерь. Значение `RMSE` означает оптимизацию корня из среднеквадратичной ошибки (что эквивалентно оптимизации MSE).

## 4. XGBoost (Extreme Gradient Boosting)

XGBoost — это масштабируемая реализация градиентного бустинга над решающими деревьями, ориентированная на скорость и качество. Основные идеи:

- Используется аддитивная композиция деревьев $\hat{y}_i = \sum_{k=1}^{K} f_k(\mathbf{x}_i)$, где каждое $f_k$ принадлежит функциональному пространству деревьев.
- Для обучения применяется оптимизация с помощью второй производной функции потерь (разложение в ряд Тейлора до второго порядка), что позволяет эффективно учитывать произвольные дифференцируемые функции потерь.
- В целевую функцию явно вводится регуляризационный член, штрафующий сложность дерева (число листьев и норму весов листьев), что помогает предотвратить переобучение.
- Реализованы методы работы с пропущенными значениями (sparsity-aware split finding), кэширование градиентов, приближённый поиск разбиений на основе гистограмм и параллельная обработка.

XGBoost широко применяется благодаря высокой производительности, устойчивости к переобучению и лидерству на соревнованиях по машинному обучению.

- **n_estimators:** Число раундов бустинга (количество деревьев). Установлено в `100`.
- **learning_rate:** Темп обучения (eta). Значение `0.1` уменьшает вклад каждого нового дерева, делая обучение более устойчивым.
- **max_depth:** Максимальная глубина одного дерева. Значение `8` ограничивает сложность каждого базового ученика.
- **subsample:** Доля случайно отбираемых строк (наблюдений) для построения каждого дерева. Значение `0.8` означает, что каждое дерево обучается на случайных 80% данных, что снижает дисперсию.
- **colsample_bytree:** Доля признаков, случайно выбираемых для каждого дерева. Значение `0.8` соответствует тому, что на каждом раунде используются 80% столбцов, повышая разнообразие деревьев.

## 5. Специализированная нейронная сеть (Custom Neural Network)

Под специализированной нейронной сетью понимается многослойная искусственная нейронная сеть с архитектурой, спроектированной для конкретной задачи. В общем виде модель представляет собой композицию нелинейных преобразований:

$$
\mathbf{h}^{(l)} = \sigma^{(l)}\left( \mathbf{W}^{(l)} \mathbf{h}^{(l-1)} + \mathbf{b}^{(l)} \right), \quad l = 1, \dots, L,
$$

где $\mathbf{h}^{(0)} = \mathbf{x}$ — входной вектор, $\mathbf{W}^{(l)}$ — весовые матрицы, $\mathbf{b}^{(l)}$ — векторы смещений, а $\sigma^{(l)}$ — выбранные активационные функции (ReLU, сигмоида, tanh и др.). Выходной слой задаёт распределение целевой переменной (линейный — для регрессии, сигмоида — для бинарной классификации, softmax — для многоклассовой).

Обучение осуществляется путём минимизации функции потерь (например, MSE или кросс-энтропии) методом обратного распространения ошибки в сочетании с градиентными оптимизаторами (SGD, Adam). Архитектура считается «специализированной» (custom), если количество скрытых слоёв, число нейронов в них, типы связей (полносвязные, свёрточные, рекуррентные) и способы регуляризации (dropout, Batch Normalization) подбираются под специфику данных и задачу.

Нейронные сети универсальны в смысле аппроксимации произвольных функций, но требуют больших объёмов данных, значительных вычислительных ресурсов и тщательной настройки архитектуры и гиперпараметров.

- **amount_of_hidden_layers:** Количество скрытых слоёв в сети. Установлено равным `5` — сеть состоит из 5 полносвязных слоёв между входом и выходом.
- **neurons_l1 … neurons_l5:** Число нейронов в каждом скрытом слое. Последовательно заданы значения `512, 512, 256, 64, 16` — постепенное сужение к выходу для выделения наиболее важных признаков.
- **activation_l1 … activation_l5:** Функции активации для каждого скрытого слоя. Все установлены в `relu` (Rectified Linear Unit) — нелинейность, ускоряющая обучение и уменьшающая проблему исчезающего градиента.
- **epochs:** Максимальное количество эпох обучения. Установлено в `50`; обучение может остановиться раньше при срабатывании early stopping.
- **batch_size:** Размер мини-пакета (количество примеров, по которым считается градиент за один шаг). Значение `16` — небольшие пакеты для более частого обновления весов.
- **optimizer:** Алгоритм оптимизации. `adam` — адаптивный метод стохастического градиентного спуска, комбинирующий идеи моментов и масштабирования градиентов.
- **learning_rate:** Начальная скорость обучения оптимизатора. Установлена в `0.001`, что типично для Adam.
- **loss_function:** Функция потерь. `mse` (среднеквадратическая ошибка) используется для задачи регрессии.
- **validation_split:** Доля обучающих данных, выделяемая в валидационную выборку для контроля переобучения. Значение `0.2` оставляет 20% данных для валидации на каждой эпохе.
- **dropout_rate:** Вероятность временного отключения нейрона во время обучения (dropout). Значение `0.3` означает, что на каждой итерации 30% нейронов скрытых слоёв случайным образом зануляются, что служит регуляризацией.
- **kernel_initializer:** Способ инициализации весовых матриц. `he_normal` — инициализация, оптимизированная для слоёв с ReLU-активацией, сохраняет дисперсию активаций.
- **patience:** Количество эпох без улучшения качества на валидационной выборке, после которого обучение останавливается (early stopping). Установлено в `20`; если метрика не улучшается 20 эпох подряд, обучение прекращается (не дожидаясь исчерпания 50 эпох).
